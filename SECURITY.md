# Security Policy

Teami is a source-visible portfolio preview. Security reports are
handled on a best-effort basis; no response SLA is promised.

## Reporting A Vulnerability

Use GitHub private vulnerability reporting for this repository. Do not include
sensitive details, credentials, exploit steps, private URLs, or tenant data in a
public issue, pull request, discussion, or comment.

If the private vulnerability reporting button is unavailable, open a general
feedback issue that asks for the private reporting channel to be enabled, but do
not include the vulnerability details there.

## Scope

Reports are in scope when they affect the current public Teami source
tree, Linear OAuth setup, local gateway polling, ambient GitHub proposal path,
local runner behavior, or documented setup process.

Out of scope for now: requests for bounty payments, support questions, feature
requests, social engineering, physical attacks, denial-of-service testing, and
findings that require access to non-public systems.
